# 公益活动模块 PRD

## 1. 概述

### 1.1 产品背景
骑闯天路骑行平台公益活动模块，为公益骑行活动提供信息展示与报名参与渠道，传递骑行正能量。

### 1.2 目标用户
- 公益骑行活动参与者
- 公益组织/合作方

### 1.3 核心价值
- 公益活动信息传播
- 在线报名参与
- 公益合作伙伴展示

---

## 2. 功能架构

### 2.1 用户端功能（微信小程序）

#### 2.1.1 公益活动列表
- 公益活动卡片展示：封面、名称、时间、地点
- 标签分类：新上线、热门、推荐
- 往期公益活动回顾入口

#### 2.1.2 公益活动详情
- 活动封面与基本信息
- Tab 页切换：
  - **活动详情**：活动介绍、目标、行程安排
  - **合作伙伴**：赞助商、公益组织展示
  - **往期回顾**：历史活动照片、数据

#### 2.1.3 公益活动报名
- 与普通活动报名流程一致
- 报名表单：个人信息、紧急联系人、附加选项

### 2.2 管理端功能

- 公益活动管理入口（与活动模块共用管理功能）
- 分类筛选时支持"公益"类别

---

## 3. 数据模型

### 3.1 CharityActivity（公益活动）
继承 Activity 数据模型，category 固定为"公益"

| 字段 | 类型 | 说明 |
|------|------|------|
| partners | string[] | 合作伙伴列表 |
| partnerLogos | string[] | 合作伙伴 LOGO |
| pastEvents | string[] | 往期活动ID列表 |
| charityType | enum | 环保/助学/扶贫/健康 |

---

## 4. 页面原型说明

### 4.1 小程序 公益活动列表页
- 入口：从首页"公益"Tab 或独立菜单进入
- 公益活动卡片列表
- 底部：往期回顾入口

### 4.2 小程序 公益活动详情页
- 活动封面大图
- 基本信息区
- Tab 切换区
- 合作伙伴展示区（图片 LOGO 排列）
- 报名按钮

---

## 5. 业务流程

### 5.1 参与公益活动流程
```
用户浏览公益活动 → 查看活动详情 → 了解合作伙伴和往期成果 →
点击报名 → 填写报名信息 → 支付（如有费用） → 报名成功
```

---

## 6. 非功能性需求

### 6.1 展示
- 公益活动页面需体现公益属性与品牌形象
- 合作伙伴 LOGO 高质量展示
- 往期活动照片画廊

### 6.2 信任
- 公益活动需展示真实性认证标识
- 资金用途透明（如适用）
- 活动成果公示

---

## 7. 依赖关系

- 依赖活动模块（报名、支付通用能力）
- 依赖用户模块（用户信息）
