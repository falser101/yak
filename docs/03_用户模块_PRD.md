# 用户模块 PRD

## 1. 概述

### 1.1 产品背景
骑闯天路骑行平台用户模块，管理平台用户账户体系，包括用户注册登录、个人信息管理、会员等级等功能。

### 1.2 目标用户
- 骑行爱好者（普通用户）
- 平台管理员

### 1.3 核心价值
- 安全的用户认证体系
- 统一的用户信息管理
- 会员等级与权益体系

---

## 2. 功能架构

### 2.1 用户端功能（微信小程序）

#### 2.1.1 用户登录
- 微信一键登录（主要方式）：通过 wx.getUserProfile 获取用户信息
- 手机号授权登录：通过 wx.getPhoneNumber 获取加密手机号

#### 2.1.2 个人中心
- 用户头像、昵称展示
- 会员等级标识
- 功能菜单入口：
  - 我的报名
  - 我的租车订单
  - 我的车辆收藏
  - 我的俱乐部
- 设置入口

#### 2.1.3 用户信息管理
- 头像上传/修改
- 昵称修改
- 手机号绑定/修改
- 密码修改

### 2.2 管理端功能

#### 2.2.1 用户管理（Admin Vue）
- 用户列表：头像、昵称、手机号、会员等级、注册时间、最近登录时间、累计骑行里程、状态
- 筛选：会员等级、状态
- 搜索：手机号、昵称
- 新建用户
- 编辑用户
- 启用/禁用用户

#### 2.2.2 用户管理（Admin Static）
- 用户列表
- 用户详情查看
- 状态切换

---

## 3. 数据模型

### 3.1 User（用户）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 用户ID |
| name | string | 昵称 |
| phone | string | 手机号 |
| avatar | string | 头像URL |
| membershipLevel | enum | 普通会员/银卡会员/金卡会员/钻石会员 |
| signups | number | 累计报名次数 |
| bikes | string[] | 收藏的车辆ID列表 |
| clubs | string[] | 加入的俱乐部ID列表 |
| totalRides | number | 累计骑行里程（公里） |
| regTime | datetime | 注册时间 |
| lastLoginTime | datetime | 最近登录时间 |
| status | enum | 正常/禁用 |

---

## 4. 页面原型说明

### 4.1 小程序 个人中心页
- 用户信息卡片：头像、昵称、手机号、会员等级标签
- 快捷操作区：我的报名、我的订单、我的收藏、我的俱乐部
- 设置按钮（右上角）

### 4.2 管理端用户管理页
- 左侧导航栏
- 顶部统计卡片：总用户数、本月新增、活跃用户
- 筛选条件区：会员等级下拉、状态切换
- 用户表格：头像、昵称、手机号、会员等级、注册时间、累计骑行里程、状态开关

---

## 5. 业务流程

### 5.1 登录流程
```
用户打开小程序 → 点击登录 → 微信授权 → 获取用户信息 → 登录成功 → 跳转首页
```

### 5.2 用户管理流程
```
管理员查看用户列表 → 筛选/搜索用户 → 查看用户详情 →
启用/禁用用户 或 编辑用户信息
```

---

## 6. 非功能性需求

### 6.1 安全
- 短信验证码 5 分钟有效期
- 密码加密存储（bcrypt）
- 敏感操作需二次验证
- 用户数据脱敏展示

### 6.2 体验
- 验证码发送间隔 60 秒倒计时
- 登录状态 7 天自动续期
- 页面加载时显示骨架屏

---

## 7. 依赖关系

- 依赖短信服务（验证码发送）
- 被活动模块、订单模块、俱乐部模块引用用户信息
