# 俱乐部模块 PRD

## 1. 概述

### 1.1 产品背景
为骑行俱乐部提供线上展示与成员管理平台，方便骑行爱好者找到并加入志同道合的俱乐部组织。

### 1.2 目标用户
- 骑行俱乐部管理者
- 寻找俱乐部的骑行爱好者

### 1.3 核心价值
- 俱乐部信息展示与发现
- 在线加入/退出俱乐部
- 俱乐部活动聚合

---

## 2. 功能架构

### 2.1 用户端功能（微信小程序）

#### 2.1.1 俱乐部列表
- 俱乐部卡片展示：头像、名称、城市、简介、成员数
- 城市筛选（可选）
- 热门俱乐部推荐

#### 2.1.2 俱乐部详情
- 俱乐部封面与基本信息
- 数据统计：成员数、活动数、累计里程、加入人数
- 俱乐部简介
- 近期活动列表
- 加入俱乐部按钮（已加入则显示"已加入"）

#### 2.1.3 加入/退出俱乐部
- 加入：一键申请加入
- 退出：确认后退出

#### 2.1.4 我的俱乐部
- 我加入的俱乐部列表
- 快速进入俱乐部详情

### 2.2 管理端功能

#### 2.2.1 俱乐部管理（Admin Static）
- 俱乐部列表：筛选（城市）、搜索、分页
- 创建俱乐部：名称、城市、头像、简介
- 编辑俱乐部信息
- 删除俱乐部
- 查看成员列表
- 成员管理：查看、移除成员

---

## 3. 数据模型

### 3.1 Club（俱乐部）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 俱乐部ID |
| name | string | 俱乐部名称 |
| city | string | 所在城市 |
| emoji | string | 俱乐部头像/图标 |
| description | string | 俱乐部简介 |
| memberCount | number | 成员数量 |
| eventCount | number | 累计活动数 |
| totalKm | number | 累计骑行里程 |
| joinCount | number | 申请加入人数 |
| createTime | datetime | 创建时间 |
| status | enum | 正常/已解散 |

### 3.2 ClubMember（俱乐部成员）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 成员ID |
| clubId | string | 俱乐部ID |
| userId | string | 用户ID |
| userName | string | 用户昵称 |
| userPhone | string | 用户手机号 |
| joinDate | datetime | 加入时间 |
| status | enum | 正常/已退出 |
| eventCount | number | 参与活动数 |

---

## 4. 页面原型说明

### 4.1 小程序 俱乐部列表页
- 顶部：标题 + 城市筛选入口
- 俱乐部卡片列表：头像、名称、城市、简介、成员数

### 4.2 小程序 俱乐部详情页
- 俱乐部封面区
- 统计数据栏：成员数 | 活动数 | 里程
- 俱乐部简介区
- 近期活动列表
- 底部操作栏：加入俱乐部 / 已加入

### 4.3 管理端俱乐部管理页
- 左侧导航栏
- 顶部：筛选条件 + 搜索 + 新建按钮
- 表格：头像、名称、城市、成员数、活动数、操作（编辑、删除、成员管理）

### 4.4 管理端成员管理弹窗
- 成员列表表格：昵称、手机号、加入时间、参与活动数、状态
- 移除成员操作

---

## 5. 业务流程

### 5.1 加入俱乐部流程
```
用户浏览俱乐部列表 → 进入俱乐部详情 → 点击加入 → 申请成功 →
等待审核（可选） → 加入成功
```

### 5.2 俱乐部管理流程
```
管理员创建俱乐部 → 完善俱乐部信息 → 用户申请加入 →
管理员审核（可选） → 成员加入 → 俱乐部活动发布
```

---

## 6. 非功能性需求

### 6.1 安全
- 俱乐部信息需审核后上线
- 成员列表需登录后查看
- 管理员操作记录

### 6.2 体验
- 俱乐部搜索支持模糊匹配
- 加入/退出操作需二次确认
- 成员数据实时更新

---

## 7. 依赖关系

- 依赖用户模块（用户信息、登录状态）
- 依赖活动模块（俱乐部活动展示）
