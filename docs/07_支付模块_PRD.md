# 支付模块 PRD

## 1. 概述

### 1.1 产品背景
骑闯天路骑行平台支付模块，为活动报名、车辆租赁等业务提供在线支付能力，支持多种支付方式。

### 1.2 目标用户
- C端用户（支付费用）
- 平台运营方（收款、对账）

### 1.3 核心价值
- 安全的在线支付
- 多种支付方式
- 清晰的账单与订单关联

---

## 2. 功能架构

### 2.1 用户端功能（微信小程序）

#### 2.1.1 支付页面
- 订单信息汇总：订单号、商品名称、金额
- 支付方式：微信支付（主要）、到店支付
- 支付按钮（调用 wx.requestPayment）
- 支付倒计时（订单有效期）

#### 2.1.2 支付结果
- 支付成功页面：订单号、支付金额、支付方式、完成时间
- 后续操作入口：查看订单、返回首页

### 2.2 管理端功能

#### 2.2.1 订单管理（Admin Static）
- 订单列表：订单号、用户、关联业务、金额、支付方式、状态、时间
- 筛选：订单类型、支付状态、时间范围
- 导出订单
- 确认到账操作（针对到店支付）

---

## 3. 数据模型

### 3.1 PaymentRecord（支付记录）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 支付记录ID |
| orderId | string | 关联订单ID |
| orderType | enum | 活动报名/租车订单 |
| amount | number | 支付金额 |
| payMethod | enum | 微信支付/到店支付 |
| payTime | datetime | 支付时间 |
| transactionId | string | 第三方交易流水号 |
| status | enum | 待支付/已支付/已取消/已退款 |
| userId | string | 用户ID |

---

## 4. 页面原型说明

### 4.1 小程序 支付页面
- 订单信息卡片：订单号、商品描述、应付金额
- 支付方式：微信支付（默认选中）、到店支付
- 底部：确认支付按钮（显示金额）
- 温馨提示区：支付有效期、取消政策

### 4.2 小程序 支付成功页
- 成功图标动画
- 订单信息：订单号、支付金额、支付方式
- 操作按钮：查看订单详情、返回首页

### 4.3 管理端订单管理页
- 左侧导航栏
- 顶部统计：今日收入、待支付订单、已完成订单
- 筛选条件：订单类型、支付状态、日期范围
- 订单表格：订单号、用户、商品信息、金额、支付方式、状态、时间、操作

---

## 5. 业务流程

### 5.1 在线支付流程
```
用户选择支付方式 → 点击支付 → 调用第三方支付SDK →
用户完成支付 → 支付结果回调 → 更新订单状态 → 显示成功页面
```

### 5.2 到店支付流程
```
用户选择到店支付 → 提交订单 → 订单待支付状态 →
用户到店付款 → 管理员确认收款 → 订单已支付状态
```

### 5.3 退款流程
```
用户申请退款 / 管理员操作退款 → 系统处理退款 →
第三方原路退回 → 更新订单状态 → 发送退款通知
```

---

## 6. 非功能性需求

### 6.1 安全
- 支付密钥服务端存储
- 支付金额服务端校验
- 防止重复支付（幂等性）
- 支付回调签名验证

### 6.2 性能
- 支付回调处理 < 3s
- 支持支付高峰期高并发

### 6.3 可靠性
- 支付失败自动重试
- 掉单自动核对与补单
- 完整的支付日志记录

### 6.4 合规
- 符合微信支付商户接入规范
- 支持交易凭证开具
- 退款时效合规

---

## 7. 依赖关系

- 依赖微信支付商户平台
- 被活动报名模块、租车订单模块调用
