# 骑闯天路骑行平台 PRD

## 1. 产品概述

### 1.1 产品背景
骑闯天路骑行平台由**西藏骑闯第三极体育文化传播有限公司**打造，是一家专注于高原骑行体验的服务平台，以青海湖和西藏高原路线为核心，提供活动赛事、骑行装备租赁、俱乐部社区等综合服务。

### 1.2 产品定位
- **愿景**：成为国内领先的高原骑行服务平台
- **使命**：让每一位骑行爱好者都能体验天路之美
- **核心用户**：高原骑行爱好者、户外运动参与者、团队骑行活动组织方

### 1.3 产品矩阵
| 端 | 形式 | 定位 |
|----|------|------|
| 微信小程序 | WeChat Mini Program | C端用户主入口，活动报名、租车、俱乐部 |
| Admin Vue | Vue 3 SPA | 新版管理后台，现代化体验 |
| Admin Static | 静态 HTML | 轻量级管理后台，快速部署 |

### 2.1 功能模块总览

```
骑闯天路骑行平台
├── 活动模块
│   ├── 活动列表（赛事/公益/亲子/俱乐部）
│   ├── 活动详情
│   ├── 活动报名
│   └── 我的报名
├── 骑行装备模块
│   ├── 租车首页（分类筛选）
│   ├── 车型详情
│   ├── 品牌目录
│   ├── 系列目录
│   ├── 租车预订
│   └── 我的订单
├── 俱乐部模块
│   ├── 俱乐部列表
│   ├── 俱乐部详情
│   ├── 加入/退出俱乐部
│   └── 我的俱乐部
├── 公益活动模块
│   ├── 公益活动列表
│   ├── 公益活动详情
│   └── 合作伙伴展示
├── 门店模块
│   ├── 门店列表
│   └── 门店详情
├── 用户模块
│   ├── 登录/注册
│   └── 个人中心
├── 支付模块
│   ├── 支付方式选择
│   └── 支付结果
└── 管理后台
    ├── Dashboard 控制台
    ├── 活动管理
    ├── 报名管理
    ├── 车辆管理
    ├── 俱乐部管理
    ├── 订单管理
    ├── 门店管理
    └── 用户管理
```

### 2.2 用户旅程

```
新用户
  ↓
打开微信小程序 → 浏览活动/车型 → 微信授权登录 → 报名活动/租车 → 微信支付 → 完成
  ↓
成为会员 → 加入俱乐部 → 参与更多活动 → 推荐给朋友
  ↓
老用户
  ↓
打开小程序 → 查看我的报名/订单 → 管理俱乐部 → 续费/续租
```

## 3. 视觉规范（基于现有静态页面）

### 3.1 品牌色
| 用途 | 色值 |
|------|------|
| 主色 | #409EFF（Element Plus 蓝） |
| 成功 | #67C23A |
| 警告 | #E6A23C |
| 危险 | #F56C6C |
| 信息 | #909399 |

### 3.2 车型分类颜色
| 分类 | 颜色 |
|------|------|
| 山地车 | #8B4513（棕色） |
| 公路车 | #4169E1（蓝色） |
| 折叠车 | #9370DB（紫色） |
| 电动车 | #32CD32（绿色） |

### 3.3 字体
- 小程序：微信默认字体栈（主要使用 system-font）
- 管理后台：Element Plus 默认字体

### 3.4 图标
- 小程序使用 WeChat 内置图标或自定义 icon
- 管理后台使用 Element Plus 内置图标

### 3.5 小程序规范
- 遵循微信小程序设计规范
- 底部 Tab Bar 使用微信原生导航
- 页面路径最多 10 层
- 单包体积不超过 2MB

## 5. 技术架构

### 5.1 前端技术栈
| 端 | 技术栈 |
|----|--------|
| 微信小程序 | 原生开发（WXML/WXSS/JS）或 uni-app跨平台 |
| Admin Vue | Vue 3 + TypeScript + Vite + Element Plus |
| Admin Static | 原生 HTML + CSS + JavaScript |

### 5.2 微信小程序技术方案
| 层级 | 技术 |
|-----|------|
| 视图层 | WXML + WXSS |
| 逻辑层 | JavaScript |
| 组件 | WeChat 原生组件 + 自定义组件 |
| 状态管理 | MobX-globx / Redux |
| 网络请求 | wx.request / Fly.js |
| 本地存储 | wx.setStorage/wx.getStorage |
| 支付 | wx.requestPayment |

### 5.3 部署结构
```
/
├── miniprogram/    # 微信小程序源码
├── admin/          # Vue 管理后台（需构建）
├── admin-static/   # 静态管理后台（直接部署）
├── prd/            # PRD 文档
├── dev/            # 技术方案文档
└── test/           # 测试报告
```
